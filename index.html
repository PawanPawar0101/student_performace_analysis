How It Works
1

Data Collection

Student data is collected including academic, behavioral, and socio-economic factors (14 features total).

2

Preprocessing

Features are normalized using StandardScaler. Categorical variables are label encoded. Data is split 80/20 for training and testing.

3

Model Training

Random Forest with 200 trees is trained. Feature importance is computed. 5-fold cross-validation ensures reliability.

4

Prediction

Given new student inputs, the model classifies into A/B/C/D/F grades with confidence probability scores.

5

Recommendations

Based on weak feature values, the system generates personalized, actionable study recommendations.

REST API Documentation

POST /predict

Predict academic performance for a student.

POST http://localhost:5000/predict
Content-Type: application/json

{
  "age": 19,
  "gender": 1,
  "study_hours": 6,
  "attendance": 82,
  "prev_grades": 72,
  "parental_education": 2,
  "internet_access": 1,
  "extracurricular": 1,
  "sleep_hours": 7.5,
  "health": 4,
  "absences": 3,
  "tutoring": 1,
  "parental_support": 3,
  "motivation": 4
}
// Response
{
  "success": true,
  "predicted_grade": "B",
  "grade_category": "Good",
  "confidence": 78.5,
  "performance_score": 81.2,
  "recommendations": ["Keep up the good work!"],
  "feature_importance": [
    { "feature": "Study Hours", "importance": 24.5 },
    ...
  ]
}
